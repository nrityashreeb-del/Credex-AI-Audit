# Project Reflection

## 1. Challenge & Execution
The primary challenge of this project was the condensed timeline. Due to a system notification delay, I received the project brief on May 12th with a May 13th deadline. I chose to handle this ambiguity by executing a "Rapid Sprint." Instead of over-complicating the UI, I prioritized a stable, functional Audit Engine and deep-dive documentation into the Unit Economics of the tool.

## 2. Technical Learnings
Building this tool reinforced the importance of "Business-First Engineering." I learned that a front-end developer’s role isn't just to make things look pretty, but to guide the user toward a logical conclusion—in this case, identifying financial waste. Implementing the audit logic in vanilla JavaScript allowed me to focus on accuracy and performance without the overhead of external libraries.

## 3. Future Improvements
If I had the full 7-day window, I would have implemented:
- **API Integrations:** Connecting to live pricing APIs for real-time data accuracy.
- **Advanced Visualization:** Adding Chart.js to visually represent spend vs. savings through interactive graphs.
- **Expanded Scope:** Including a broader range of enterprise SaaS tools beyond just AI.

## 4. Conclusion
Despite the shortened timeframe, I am proud of the functional MVP I have delivered. It demonstrates my ability to remain disciplined under pressure, think like an entrepreneur, and ship a product that provides immediate value to the end-user and the company.
